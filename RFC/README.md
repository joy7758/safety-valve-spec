# RFC Process (SVS)

## Goal
All spec changes go through an RFC to prevent semantic drift and preserve compatibility.

## File naming
- `RFC/0001-<short-title>.md`
- Increment the number for each new proposal.

## Workflow
1) Open a PR with a new RFC file.
2) Discuss in PR comments.
3) If accepted, merge RFC first.
4) Implement changes in a follow-up PR (or same PR if small), with conformance passing.
